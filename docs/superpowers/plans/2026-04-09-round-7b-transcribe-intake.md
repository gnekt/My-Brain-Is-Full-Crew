# Round 7B Transcribe Intake Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the active `/transcribe` intake flow into a Codex-first two-layer contract so raw audio is gated immediately, transcript work starts with a minimal first layer, and `transcriber` stays aligned with the skill.

**Architecture:** Keep the dispatcher, the six processing modes, and the existing transcription output system in place. Update only the input-side contract in `skills/transcribe/SKILL.md` and the mirrored runtime wording in `agents/transcriber.md`, then add a dedicated smoke test to guard the new intake boundary.

**Tech Stack:** Markdown runtime instructions, shell smoke tests, Codex runtime docs

---

### Task 1: Replace the front-loaded interview with source-type gating

**Files:**
- Modify: `skills/transcribe/SKILL.md`

- [ ] Rewrite the current `Intake Interview` section so `/transcribe` starts by distinguishing `raw audio only` from `transcript or transcript-like text`.
- [ ] Remove the old first-turn requirement list that asks for date, processing mode, language, priority flags, and transcript format before the skill knows whether it can proceed.
- [ ] Add a first-turn raw-audio gate that immediately states the current Codex runtime cannot natively transcribe raw audio by itself.
- [ ] Keep the supported next step explicit: the user can return with transcript text generated by an external workflow and continue inside `/transcribe`.

### Task 2: Define the new two-layer transcript intake contract

**Files:**
- Modify: `skills/transcribe/SKILL.md`

- [ ] Add a fixed first-layer intake that only collects `Purpose`, `Output target`, `Destination`, and `Speaker context`.
- [ ] Rewrite the follow-up guidance so transcript-based requests expand into a second layer only after the first layer is known.
- [ ] Describe second-layer expansion as a mixed model: first by source type, then by output target.
- [ ] Replace broad default questioning with sparse, purpose-driven follow-ups so items like language, transcript format, priority flags, and timestamp handling are asked only when they materially affect output.

### Task 3: Align `transcriber` to the new intake contract

**Files:**
- Modify: `agents/transcriber.md`

- [ ] Update the core-processing section so it describes `/transcribe` as the owner of the new two-layer intake path.
- [ ] Align the raw-audio wording with the skill so the agent also reflects immediate limit disclosure instead of implying native audio transcription.
- [ ] Align transcript-path wording so the agent explicitly references the lighter first layer and adaptive second-layer expansion.
- [ ] Re-read the agent description and top-level behavior notes to confirm they no longer suggest a parallel intake model separate from `/transcribe`.
